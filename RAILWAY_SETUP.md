# Настройка Railway для работы с PostgreSQL

## Критически важно: Переменная DATABASE_URL

Для работы приложения в Railway **обязательно** нужно установить переменную окружения `DATABASE_URL`.

## Как установить DATABASE_URL в Railway:

1. Откройте ваш проект в Railway
2. Перейдите в раздел **Variables** (Переменные окружения)
3. Найдите или создайте переменную `DATABASE_URL`
4. Установите значение в формате:

```
postgresql://postgres:ВАШ_ПАРОЛЬ@switchyard.proxy.rlwy.net:40807/railway?sslmode=require
```

**ИЛИ** используйте внутренний URL для production:

```
postgresql://postgres:ВАШ_ПАРОЛЬ@postgres.railway.internal:5432/railway
```

## Проверка:

После установки переменной:
1. Railway автоматически пересоберет приложение
2. Проверьте логи Railway - не должно быть ошибок "DATABASE_URL not found"
3. Откройте приложение и проверьте работу симулятора

## Важно:

- `DATABASE_URL` должна быть установлена **ДО** деплоя
- Railway автоматически инжектирует переменные окружения в приложение
- Если переменная не установлена, приложение не сможет подключиться к базе данных

## Альтернатива:

Если у вас есть отдельные переменные (`PGHOST`, `PGPASSWORD` и т.д.), Railway может автоматически создать `DATABASE_URL`. Проверьте раздел Variables в Railway.
